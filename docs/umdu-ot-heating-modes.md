# Режимы управления отоплением: ПИД (PID) и ПЗА (Equiterm)

umdu ot поддерживает два основных подхода к управлению отоплением:

- `ПЗА (Equiterm)` — погодозависимое управление;
- `ПИД (PID)` — коррекция по ошибке между целевой и фактической температурой помещения.

Их можно использовать по отдельности или совместно.

## Как выбрать режим управления отоплением

Ориентируйтесь на такой базовый выбор:

- если хотите, чтобы отопление заранее подстраивалось под погоду, начинайте с `ПЗА (Equiterm)`;
- если главная цель — точнее удерживать температуру в комнате, смотрите в сторону `ПИД (PID)`;
- если уже настроена погодозависимая кривая, `ПИД (PID)` можно использовать как дополнительную корректировку;
- если у вас в настройках `OpenTherm` включен `Нативный режим OTC (расчёт температуры теплоносителя)`, режимы `ПИД (PID)` и `ПЗА (Equiterm)` использовать нельзя.

## ПЗА (Equiterm)

### Что это

`ПЗА (Equiterm)` — это погодозависимый режим. Он подбирает температуру отопления с учетом наружной температуры и помогает сделать работу котла более плавной.

Для `ПЗА (Equiterm)` обязателен датчик `#1 Внешняя температура`. Без него включается аварийный режим. Датчик `#2 Внутренняя температура` для базовой работы `ПЗА (Equiterm)` не обязателен и используется только для дополнительной коррекции.

В качестве наружной температуры можно использовать:

- проводной датчик;
- `BLE`-датчик;
- значение, передаваемое из Home Assistant через внешний источник температуры.

### Где включается

- сами настройки доступны в веб-интерфейсе в `Настройки ПЗА`;
- через Home Assistant, если включены `MQTT` и `Home Assistant Discovery`.

::: info Примечание
`ПЗА (Equiterm)` можно использовать с настройками по умолчанию. Для начала работы достаточно просто активировать режим.
:::

### Как настраивать

Для подбора коэффициентов удобно использовать [Excel-таблицу](https://github.com/umduru/OTGateway/blob/umdu_ot/assets/equitherm_calc.xlsx). В нее можно ввести температуру внутри и снаружи дома, подобрать коэффициенты и на графике увидеть, какую температуру установит котел.

Основные коэффициенты:

- `N` — коэффициент кривой нагрева. Подбирается индивидуально в зависимости от теплоизоляции помещения, отапливаемой площади и других условий. Диапазон: `0,001...10`, значение по умолчанию: `0,7`, шаг: `0,001`.
- `K` — поправка на желаемую температуру в помещении. Диапазон: `0...10`, значение по умолчанию: `3`, шаг: `0,01`.
- `T` — поправка для термостата. Диапазон: `0...10`, значение по умолчанию: `2`, шаг: `0,01`.

Рекомендуемый порядок настройки:

1. Отключите гистерезис, установив значение `0` в настройках отопления, и отключите `ПИД (PID)` в настройках ПИД-регулятора.
2. Установите коэффициенты `K` и `T` в `0`.
3. Сначала подберите кривую нагрева, то есть коэффициент `N`. Если в доме низкие теплопотери, начните с `0,5`. В остальных случаях начните с `0,7`.
4. Когда температура в доме перестанет меняться, увеличивайте или уменьшайте `N` с шагом `0,1`, пока не подберете подходящую кривую.
5. Во время подбора кривой ориентируйтесь на стабильную температуру в помещении около `20 +- 0,5` градусов при изменении наружной температуры. Этот этап небыстрый и обычно занимает `1-2 дня`. Учитывайте, что солнечный свет и другие внешние источники тепла влияют на результат.
6. После подбора кривой выберите коэффициент `K`. Он влияет на корректировку температуры теплоносителя для поддержания заданной температуры в помещении. Например, если задано `23` градуса, а в доме `20`, можно начать со значения `2` и посмотреть на результат после стабилизации.
7. Затем подберите коэффициент `T`. Для более плавного нагрева обычно подходят значения `1-5`. Для более быстрого нагрева можно использовать `6-10`, но в этом случае возрастает риск перегрева помещения.
8. В течение нескольких дней проверьте, как система работает при разных заданных температурах.
9. Если вы отключали гистерезис и `ПИД (PID)` на первом шаге, включите их обратно после завершения настройки.

### Типовые ошибки

- использовать неверный или нестабильный уличный датчик;
- подбирать параметры слишком быстро, не дожидаясь стабилизации дома;
- менять сразу несколько коэффициентов;
- одновременно пытаться настраивать и `ПЗА (Equiterm)`, и `ПИД (PID)`, не понимая вклад каждого режима.

## ПИД (PID)

### Что это

`ПИД (PID)` корректирует уставку отопления по разнице между целевой и фактической температурой в помещении.

Это полезно, когда хочется точнее удерживать комнатную температуру, а не только следовать погодной кривой.

Для `ПИД (PID)` обязателен датчик `#2 Внутренняя температура`. Без него включается аварийный режим.

### Где включается

- в web-интерфейсе модуля в настройках `ПИД (PID)`;
- через Home Assistant, если включены `MQTT` и `Home Assistant Discovery`.

::: info Примечание
`ПИД (PID)` можно использовать с настройками по умолчанию. Для начала работы достаточно просто активировать режим.
:::

::: info Home Assistant
При настройке через Home Assistant часть числовых сущностей может быть изначально отключена. В этом случае сначала включите их в реестре сущностей.
:::

### Что означают основные параметры

- `P` — насколько резко модуль реагирует на текущую ошибку температуры;
- `I` — насколько сильно учитывается накопленная ошибка за время;
- `D` — насколько сильно учитывается скорость изменения температуры;
- `DT` — как часто выполняется расчет;
- `minTemp` и `maxTemp` — ограничения результата `ПИД (PID)`;
- `Deadband` — зона около целевой температуры, где коррекция становится мягче.

### Базовый порядок настройки

1. Убедитесь, что датчик температуры помещения показывает реальное значение.
2. Начните с небольших изменений и меняйте по одному параметру за раз.
3. Сначала подбирайте `P`.
4. Затем, если нужно, добавляйте `I`.
5. `D` оставляйте напоследок и трогайте только если понимаете, зачем он нужен в вашем объекте.
6. Если система начинает заметно раскачиваться, вернитесь на шаг назад.

### Типовые ошибки

- настраивать `ПИД (PID)` без надежного комнатного датчика;
- слишком сильно увеличивать `P` и `I`;
- менять несколько коэффициентов одновременно;
- оценивать результат слишком быстро, без достаточного времени наблюдения.
